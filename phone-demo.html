a simple phone-style message display thing.


this was made for the community project to demo an idea for displaying the site's quiz in this phone-messaging style format. https://github.com/jinritv/Streamer-match-makers/

the original idea was that we change or add buttons at the bottom of the phone to answer each question, but for this example i've just hardcoded the questions and answers. just press the button to move on to the next.

i thought the phone-style concept was interesting at first, but now that i see it, seems like it'd feel weird having it on a desktop website, and doesn't really fit the vibe of the twiri site so far. so, we probably won't end up using it for the interaction with Twiri anyways.

i hid the scrollbar but it should still scroll with the scrollwheel, provided there are enough messages on the screen to overflow the view. btw i've only tested it with firefox.

the messages are read as pairs: a send message and a response message. once you send a message, there will be a short delay before seing the 'person is typing' message. it will show for a few seconds to imitate typing/thinking/whater. then it will display the other message, and let you send another one.
